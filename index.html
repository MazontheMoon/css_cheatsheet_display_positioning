What's this all about?

So far in my collection of cheatsheets have we have looked at topics such as the Box Model and Selectors which are about about how the content boxes work and methods to select them. Now we are moving on to where we want to put the content boxes in our document now that we know how to select them.

Display

Everything is a box and the type of box applied to an element is specified by the display property.

  • Block
  • Inline
  • Inline-Block
  • Flex
  • Grid
  • None
Block
display:block;

Stacks open other boxes with a new line for each block

Can specify width and height

Inline
display: inline;

Flows with the documents content (not taking a new line)

Cannot specify width and height

Inline-Block
display: inline-block;

This is a mix of inline and block (not that you would have guessed from the name and flows with the documents content without creating line breaks

Can specify width and height

Flex
display: flex;

Flexbox is what you will use for 1D layouts (a row or column) such as nav menu bars

Flex is very flexible with many options and deserves a special page all to itself which I will do next and link back here!

Grid
display: grid;

This is what you will use for 2D layouts such as organising the page (and so easy to make changes for responsiveness)

Grid is also so fantastic that it deserves a special page all to itself which I will also do next and link back here!

None
display: none;

This removes the element from the flow of the document..and unlike changing it to invisible the space it takes up is also removed instead of staying as a blank spot

Positioning

Display is used to position the box for the element in the body of the document and Position is used to specify where an element is positioned within it's container box.

Position values are:

  • Static
  • Relative
  • Fixed
  • Absolute
  • Sticky

The elements are then positioned using top, right, bottom and left values (can also use clip)

Static
position:static;

Static is the default position and goes with the flow of the page and ignores top, right, etc.

Relative
position:relative;

The element is positioned relative to its normal position using top, bottom, right, or left.

Other content is not adjusted to fit any gap by moving this element

Fixed
position:fixed;

The element is positioned relative to the viewport and stays in the same place even when the page is scrolled.

It does not leave a gap from where it was originally positioned

Absolute
position:absolute;

The element is positioned relative to the nearest positioned ancestor.

The ancestor cannot be using the default static position but must explicitly set a position for this to work

Sticky
position:sticky;

The element is positioned based on the users scroll position.

It toggles between relative and fixed, is relative until a given offset is met in viewport and then is sticky.

Z-Index

When elements are positioned they can overlap.

Z-index is used to specify the stack order of elements (higher value on top and can also use -1)

Float

Float and Clear go hand in hand with each other.

The float property specifies how an element should float and the clear property specifies what elements can float beside the cleared element and on which side.

float: left; clear: left;

Itâ€™s simplest use is to wrap text around images....but this was before flex and grid came along so will see on older sites but probably not on any newer projects

Float values are:

  • left
  • right
  • none
  • inherit

Clear values are:

  • left
  • right
  • both

Clearfix

If an element is taller than the element containing it, and it is floated, it will overflow outside of its container.

One way to fix it is with this:

.clearfix { overflow: auto; }

... and it works well as long as you are able to keep control of your margins and padding (else you might see scrollbars). Instead it is much better to do it this way:

.clearfix::after { content: ""; clear: both; display: table; }

Overflow

When the size of a box is set with absolute values the content may not fit within the allowed size and will overflow.

overflow: auto;

The overflowing content is hidden and scroll bars are shown

overflow: hidden;

The overflowing content is hidden

overflow: visible;

The overflowing content is shown outside the box (default value)

overflow-x;

The overflowing content has a scroll bar for the x-axis

overflow-y;

The overflowing content has a scroll bar for the y-axis

And if you haven't had your coffee today the x-axis is right to left and the y-axis is up and down
